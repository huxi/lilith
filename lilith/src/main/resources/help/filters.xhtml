Custom Groovy filters.

It is possible to create complex filters in Groovy. The scripts must be created in "[lilith.home]/filters" and must have a ".groovy" extension.

By default, [lilith.home] equals [user.home]/.lilith but this can be changed in preferences.

A filter script receives an event-wrapper as the environment variable "input" and is expected to return either true or false.

The scripts can then be used by entering "#groovy#<filename without extension>" into the search box. After entering the "#groovy#"-prefix, the color of the textfield will indicate if a script of the given name exists.

Example 1: Show only odd events.

input.localId % 2 == 1

Example 2: Show events having a threadName that's starting with "main"

def threadName = input?.event?.threadName;
if(threadName)
{
  return threadName.startsWith("main");
}
return false;
