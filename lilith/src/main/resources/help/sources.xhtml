Sources, Source-Names & Source-Lists

Sources

Sources are the origin of events that are received by Lilith. The typical source identifier is the IP address of the machine sending the event. This would be "127.0.0.1" in case of your own computer.

Source-Names

It's possible to give a certain source a name, e.g. the source "127.0.0.1" would likely be named "localhost", using "View" => "Edit source nameâ€¦". The shortcut for this functionality is *command*-B.
Named sources are used in two places:

  • They are shown in the title of a view window so it's easier to see where the events are originating from. Names better suited for this than simply ip addresses.
  • The can be added to source lists which can then be used to either white- or black-list the receiving of events from that sources.

Named sources can be configured in Sources preferences.

Source-Lists

Source-Lists can be used to either white- or black-list on the contained sources.
In case of white-listing, only events from sources contained in the selected source list are accepted.
In case of black-listing, only events from sources not contained in the selected source list are accepted.

This functionality is mostly useful if you constantly receive events from a large amount of different applications and would like to temporarily work with only a subset of these.

You could, for example, create a source list that contains only "127.0.0.1" and white-list on that source list to receive only events originating from your own computer.

Source lists can be configured in Source Lists preferences while black- and whitelisting can be configured in Source Filtering preferences.
